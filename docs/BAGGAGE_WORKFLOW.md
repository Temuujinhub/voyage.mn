# Ачаатай зорчигчид үйлчлэх дараалал (Check-in counter)

Ачаатай ирсэн зорчигчид counter-ийн ажилтан (agent) дараах дарааллаар үйлчилнэ.

## 1. Зорчигчийг олох

- **Check-in бүртгэл** хуудсыг нээнэ — таны буудлын (UB / OT) идэвхтэй нислэг автоматаар сонгогдоно.
- Дээрх хайлтад зорчигчийн **нэр / SAP ID / PNR / утасны дугаараар** хайна, эсвэл нислэгийн жагсаалтаас олно.
- Зорчигч manifest-д байгаа эсэх, статус **PENDING** (бүртгэгдээгүй) болохыг шалгана.

## 2. Ачааг жинлэх, бүртгэх

- Зорчигчийн мөрөн дээрх **Check-in** товчийг дарж бүртгэлийн цонхыг нээнэ.
- Ачаа бүрийг жинлүүрт тавьж, жинг **кг-аар** оруулна («Ачаа нэмэх» товчоор мөр нэмнэ).
- Систем үнэгүй нормыг (үндсэн тохиргоо: **15 кг**) автоматаар шалгана:
  - Норм хэтэрсэн бол **илүү кг × төлбөр** (тохиргоогоор кг тутамд 2500₮) автоматаар бодогдоно.
  - Илүү төлбөрийг зорчигчоос авч баталгаажуулна (fee_paid).
- Шаардлагатай бол суудлыг гараар сонгоно — үгүй бол автомат хуваарилалт хийгдэнэ.

## 3. Check-in баталгаажуулах

- **«Check-in баталгаажуулах»** товчийг дарна. Систем:
  - Суудал хуваарилж, boarding pass-ийн дараалал (SEQ) олгоно;
  - Ачаа бүрд **IATA Res.740 стандартын бирк** (0 + 3 оронтой код + 6 оронтой серийн дугаар) үүсгэнэ;
  - BCBP QR кодтой **e-boarding pass** үүсгэнэ (зорчигчийн утсанд мөн харагдана).

## 4. Бирк хэвлэж, ачаанд наах

- Boarding pass цонхонд «**N ширхэг ачааны бирк бэлэн**» гэсэн мэдэгдэл гарна.
- **Хэвлэгч сонгоно** — тухайн counter-д суулгасан хэвлэгчүүдээс (Тохиргоо → «Бирк / Boarding pass хэвлэгч» хэсэгт админ урьдчилан суулгасан байна). Сонголт хөтчид хадгалагдана.
- **«Бирк хэвлэх»** товчийг дарахад 470×51мм биркнүүд сонгосон хэвлэгч рүү гарна.
- Хэвлэсэн биркийг ачаа бүрийн бариул дээр наан, **claim stub**-ийг зорчигчид өгнө.
- Ачааг тээшний хамт хөдөлгөөнд шилжүүлнэ.

## 5. Boarding pass олгох

- Зорчигч утсаараа e-boarding pass авсан бол хэвлэх шаардлагагүй.
- Хүсвэл **«Boarding pass хэвлэх»** товчоор цаасан хувийг хэвлэж өгнө.

## 6. Дараагийн шатууд (автомат)

- **Security:** зорчигчийн QR-ийг скан хийхэд статус SECURITY_PASSED болно.
- **Gate:** QR-ийг дахин скан хийж онгоцонд суулгана (BOARDED) — Gate дэлгэц бодит цагт шинэчлэгдэнэ.
- Зорчигч ирээгүй бол **Offload** — ачааг нь мөн буулгах ёстойг систем сануулна.

## Онлайн check-in хийсэн, ачаатай зорчигч (bag drop)

Зорчигч өөрөө онлайнаар бүртгүүлэхдээ «ачаатай» гэж зарласан бол `baggage_pending` төлөвтэй ирнэ:

1. Counter дээр зорчигчийг хайж олоход **ачаа хүлээгдэж буй** тэмдэглэгээ харагдана.
2. Ачааг жинлэн бүртгэж (2-р алхам), бирк хэвлэн наана (4-р алхам).
3. Boarding pass дахин олгох шаардлагагүй — зорчигчийн утсан дахь QR хүчинтэй хэвээр.
